# Template Method 패턴 Q&A 답변

1. **템플릿 메서드와 전략 패턴의 차이는?**  
   템플릿 메서드는 상위 클래스가 전체 알고리즘 순서를 고정하고, 하위 클래스가 특정 단계를 구현한다. 전략 패턴은 알고리즘 전체를 교체 가능한 객체로 제공해 컨텍스트가 적절한 전략을 선택한다.

2. **Riverpod에서 템플릿을 family로 만들면 어떤 장점이 있는가?**  
   동일한 템플릿 로직을 유지한 채, 데이터셋이나 옵션에 따라 다른 Provider 인스턴스를 생성할 수 있어 상태 충돌 없이 여러 화면에서 재사용할 수 있다.

3. **훅 메서드를 필수 구현으로 강제하려면 어떻게 해야 하는가?**  
   해당 메서드를 `abstract`로 선언하거나, 기본 구현에서 `UnsupportedError`를 던지도록 해 구현을 강제한다. 필요 시 `@mustCallSuper`를 통해 상위 로직 호출을 강제할 수도 있다.

4. **비동기 작업을 포함하려면 구조를 어떻게 바꿔야 하는가?**  
   템플릿 메서드 반환 타입을 `Future<List<String>>`처럼 비동기로 변경하고, Riverpod에서는 `FutureProvider`와 결합해 UI에서 로딩·에러 상태를 처리한다.

5. **템플릿이 너무 복잡해지면 대안은 무엇인가?**  
   작은 템플릿으로 나누거나, 파이프라인/컴포지터블 함수 조합, 혹은 전략 패턴을 이용해 단계별 전략을 주입하는 혼합 패턴으로 전환한다.
