# Strategy 패턴 Q&A 답변

1. **전략 패턴과 상태 패턴의 차이는?**  
   전략 패턴은 외부에서 알고리즘을 선택해 컨텍스트가 위임하도록 만들며, 상태 패턴은 객체 내부 상태 전환에 따라 행동이 변하도록 한다. 전략은 교체 가능한 알고리즘 패밀리에 초점을 맞추고, 상태는 전이 규칙과 상태별 책임에 초점을 둔다.

2. **전략 객체 생성을 팩토리로 감싸야 하나?**  
   전략 종류가 많거나, 생성 시 추가 의존성이 필요하다면 팩토리를 도입해 책임을 분리하는 것이 좋다. 단순히 `const` 생성자로 충분하다면 직접 생성해도 무방하다.

3. **Provider 대신 ChangeNotifier를 써도 되나?**  
   가능하지만 ChangeNotifier는 상태 변경 시 전체 소비자가 리빌드될 수 있어 제어가 어렵다. 전략 패턴처럼 값 자체를 교체하는 경우 `StateProvider`처럼 값 단위로 추적되는 Provider 계열이 더 적합하다.

4. **전략 간 공통 설정은 어디서 관리해야 하나?**  
   추상 클래스의 `protected` 메서드나 믹스인을 통해 공통 도우미를 제공하라. 이렇게 하면 중복을 줄이고 새 전략 추가 시 필요한 훅만 구현하면 된다.

5. **Flutter 테마에서 컬러 충돌을 예방하려면?**  
   `ColorScheme.fromSeed`를 사용해 색상 조합을 일관되게 관리하고, 텍스트·아이콘 대비를 `ThemeData`에서 자동으로 조정하게 둔다. 접근성 검증용 툴로 대비 비율을 체크하면 추가 안전장치를 마련할 수 있다.
